Path maps

One mapping per line: sonarr_path:local_path.
Sonarr reports series paths using its own internal mount (e.g. /tv). If Parsarr's container mounts your library at a different path (e.g. /srv/media/tv), add a line like /tv:/srv/media/tv so files land in the right place.

Sonarr webhook setup

In Sonarr, go to Settings → Connect → Add Connection → Webhook and set:

TriggersOn Grab only
MethodPOST

URL — use the base address Sonarr can use to reach Parsarr (from Sonarr’s network view, which may differ from your browser).

  • Typical Docker (same compose/network as Parsarr):
    http://parsarr:{{ settings.port }}/webhook/sonarr/grab — parsarr is the service/container name; port is Parsarr’s listen port inside the container.
  • Otherwise:
    http://<host>:<port>/webhook/sonarr/grab — replace with whatever hostname or IP and port Sonarr should use to reach Parsarr (no single value fits every setup).

Parsarr intercepts the release at grab time — before anything has been downloaded — so it can reroute problematic releases before Sonarr gets first access to the raw files.
